# Requirements Management

This project uses `uv` for dependency management while also maintaining traditional requirements files for compatibility.

## Files Overview

- `pyproject.toml` - Main project configuration with dependencies
- `uv.lock` - Lock file generated by uv with exact versions
- `requirements-pinned.txt` - Production dependencies (autogenerated from uv.lock)
- `dev-requirements.txt` - Development dependencies (autogenerated from uv.lock)
- `requirements-build.txt` - Build dependencies (compiled from requirements-pinned.txt + requirements-build-extra.txt, no `-e .`; app is installed via `pip install .`). Used by Docker and Hermeto/Cachi2.
- `requirements-build-extra.txt` - Build-time-only deps (e.g. pytest-runner, setuptools-scm, wheel) so Cachi2 prefetches them for hermetic builds

## Working with Dependencies

### Using uv (Recommended)

```bash
# Install all dependencies
uv sync

# Install with development dependencies
uv sync --extra dev

# Add a new dependency
uv add package-name

# Add a development dependency
uv add --dev package-name

# Update dependencies
uv lock
```

### Using traditional pip

```bash
# Install production dependencies
pip install -r requirements-pinned.txt

# Install development dependencies
pip install -r dev-requirements.txt

# Install build dependencies and the app
pip install -r requirements-build.txt
pip install .
```

## Syncing Requirements Files

The `requirements-pinned.txt`, `dev-requirements.txt`, and `requirements-build.txt` files are automatically generated from `uv.lock`. To update them:

### Manual Sync

```bash
# Using the script directly
./sync-requirements.sh

# Using make
make sync-requirements
```

### Check if files are in sync

```bash
make requirements-check
```

## Automated Sync

The requirements files are automatically synced via GitHub Actions:

- **On push to devel**: Files are automatically updated and committed
- **On pull requests**: The workflow checks if files are in sync and fails if they're not

If you see a PR check failure, run `./sync-requirements.sh` locally and commit the updated files.

## File Contents

- `requirements-pinned.txt`: Contains only production dependencies (no dev dependencies)
- `dev-requirements.txt`: Contains development dependencies plus any shared dependencies they need
- `requirements-build.txt`: Contains build dependencies compiled from requirements-pinned.txt

All files include:

- Exact version pins
- Security hashes for verification
- Comments showing dependency relationships
- Auto-generated headers indicating their source
